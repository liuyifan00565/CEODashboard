# CEO AI客服销售驾驶舱需求文档

## 1. 项目背景

成都福客人工智能公司需要搭建一个 CEO 视角下的 AI 客服销售经营驾驶舱。当前阶段先做 HTML Demo 给 CEO 过目，用于选择看板框架和核心经营指标，不接真实数据。

本次修订后的核心反馈是：页面颗粒度不能太细，最小只关注“月”维度，排版需要更简洁、更直观。

## 2. 看板定位

看板定位为 CEO 月度经营驾驶舱，不是销售过程管理页面，也不是功能堆砌型 BI 页面。

CEO 首屏只需要快速回答四个问题：

- 本月卖了多少钱。
- 本月目标完成到什么程度。
- 总投入和费比是否在合理范围。
- 续费率是否健康，是否被某个渠道或购买版本拖低。

财务健康（已回款、应收、广告费、人力成本、ROI）不再放在顶部四个核心卡片里，改到主体区域用柱状图展示，避免首屏 KPI 判断过多。

## 3. 时间颗粒度

可以通过切换 年月日 纬度，并且可以通过日历选择器选择时间范围

页面不再展示：

- 日级走势，例如 06-01、06-05、06-10。
- 本月第几天的日进度拆解。
- 日维度累计折线。
- 城市级明细列表。

页面保留：

- 本月
- 上月
- 月度环比
- 年度累计
- 月度目标聚合到季度、年度

## 4. 交付形式与技术架构（2026-06-24 重大调整）

本节于 2026-06-24 经 CEO 确认，**取代原「单文件 HTML Demo」方案**。

- 形式：**React + Vite** 前端项目（不再是单文件 HTML，需要构建/部署）。项目目录：`cockpit/`。
- 目标浏览器：**仅 Chrome / Chromium 内核**，不做 Safari、Firefox 降级。
- 性能策略：CEO 设备性能充足，**优先追求极致视觉体验**，不为性能牺牲效果（不再要求「先渲染再异步降级」）。
- 数据：前端内置模拟数据，暂不接真实数据。
- 图表：**ECharts**（React 中用项目内 `EChart` 组件封装）。
- 动效：**GSAP** + **ReactBits** 组件。
- 3D / 玻璃质感：**three.js + @react-three/fiber + @react-three/drei + maath**。
- 品牌：使用项目内 `logo.png`。
- 适配：电脑浏览器一屏展示。

### 4.1 组件选型（优先用现成组件库，不自造轮子）

| 用途 | 选型 | 来源 |
| --- | --- | --- |
| 页面背景 | ReactBits **DotField** | `@react-bits/DotField-JS-CSS` |
| 玻璃鼠标指针 / 开场玻璃 | ReactBits **FluidGlass** | 官方 R3F 组件（three + fiber + drei + maath） |
| 搜索框 | ReactBits **GlassSurface**（**展开式**） | `@react-bits/GlassSurface-JS-CSS` |
| 日期 / 范围选择器 | Flatpickr 行为 + ReactBits **GlassSurface** 外壳 | `flatpickr` + `@react-bits/GlassSurface-JS-CSS` |
| 年/月/日维度筛选 | ReactBits **GlassSurface** 外壳 + 自定义滑动 thumb | `@react-bits/GlassSurface-JS-CSS` |
| 纵向侧边菜单、其余 UI 动效 | 优先 ReactBits / 成熟组件库 | 统一黑白灰极简风格 |

> 安装方式：ReactBits 组件经 shadcn registry 安装（`npx shadcn@latest add @react-bits/<Component>`），或直接取 registry JSON（`https://reactbits.dev/r/<Component>.json`）落地到 `src/components/`。

## 5. 展示范围

### 5.1 展示内容

业务侧展示销售业绩：

- 月度回款和目标
- 年度回款和目标
- 完成率
- 订单数
- 客单价
- 月度环比
- 续费率（需支持按渠道和购买版本拆分）

财务侧展示内部财务：

- 已回款
- 应收 / 未回款
- 总投入费用（线上广告费和人力成本）
- ROI
- 渠道 ROI（渠道回款 / 渠道投入）

财务侧不作为顶部 KPI 卡片，放在主体区域以柱状图方式展示。

## 6. 渠道与战区

销售渠道（共四个）：

- 线上
- 线下（分为华南线下、华东线下）
- 代理

线下渠道分战区：

- 华南战区：广州、普宁
- 华东战区：杭州、武汉

展示层面只展示到战区，不在首屏展开城市明细。城市信息仅作为战区归属背景。

## 7. 产品版本

展示三个 AI 客服版本：

| 版本 | 价格 |
| --- | ---: |
| 启航版 | 16,800 元 |
| 卓越版 | 39,800 元 |
| 至尊版 | 99,800 元 |

版本模块展示：

- 月度销售套数
- 月度回款金额
- 月度环比
- 续费率筛选维度：续费率二级卡片需要支持全部版本、启航版、卓越版、至尊版筛选，判断续费率是否因购买版本不同而分化。

## 8. 目标管理

年度目标暂按 5,800 万展示。

## 9. 环比与颜色口径

环比口径：

- 本月真实数据 vs 上月真实数据。

颜色口径：

- 荧光粉表示上涨 / 预警。
- 荧光绿 `rgb(223, 255, 0)` 表示下跌 / 目标完成优秀。
- 普通文字和普通控件使用黑白，不使用灰色或其它彩色。

预警口径：

- 月度目标未完成时显示预警。
- 渠道完成率明显落后时显示预警。
- 应收金额偏高时显示财务压力。

## 10. 页面结构


### 10.1 顶部栏

包含：

- Logo
- 看板标题
- 月度视角说明
- 搜索框

另外需要左侧有个菜单栏，需要预留好可以切换查看细类纬度数据的入口
- 经营总览
- 渠道分析
- 财务分析
- Token分析

另外需要日历时间段选择器，以及年月日时间纬度筛选

### 10.2 核心 KPI 行

只保留四个核心卡片：

- 本月回款和目标完成情况、以及目标缺口
- 年度累计回款和年度目标完成情况、以及目标缺口
- 总的投入成本和费比（总投入费用/赢单金额）
- 续费率（到期客户、已续客户、续费金额、较上期变化）

这四个卡片是 CEO 第一眼要看的内容。

以上四个卡片是总览的数据，需要支持点击卡片后，卡片翻转并放大，可以选择四种渠道和时间纬度，显示 年月日 纬度的柱状图，并且柱子支持点击，点击后卡片数据联动变化，并且需要显示环比数据（时间如果选了年就是年度环比、月就是月度环比、日就是日度环比，环比的涨跌用股市的标识，红涨绿跌）
1. 小卡片执行翻转 / 放大动效。
2. 小卡片展开为居中悬浮的大数据卡片，背景保留驾驶舱上下文。
3. 续费率卡片展开后，柱状图横轴展示各渠道续费率；筛选控件需要增加购买版本筛选（全部版本 / 启航版 / 卓越版 / 至尊版），点击渠道柱子后展示该渠道到期客户、已续客户、未续客户、续费金额和较上期变化。

### 10.3 主体区域

主体区域分三块：

- 月度经营趋势：按月展示目标、回款、完成率；卡片高度需要与右侧「本月渠道完成」保持一致。
- 渠道 ROI 分析：放在月度经营趋势下方，按渠道展示 ROI、渠道回款、渠道投入。ROI 口径为「渠道回款 / 渠道投入」，用于识别投入产出偏低的渠道。
- 本月渠道完成：展示线上、线下、代理，以及线下华南、华东战区。
- 版本与财务：展示三个版本的销售结构，并在下方展示财务健康柱状图（已回款、应收、广告费、人力成本）。

### 10.4 二级数据卡片

CEO 需要查看月度环比、年度累计、渠道目标完成对比、产品版本对比、续费率按渠道和版本拆分等趋势数据，但这些内容不应全部堆在首屏。

因此首屏核心 KPI 卡片同时作为二级数据入口：

二级数据卡片用于承载环比和趋势图表；首屏仍只保留当前月经营总览和核心判断数字。

## 11. 交互需求

### 11.1 搜索

搜索框支持：搜索关键词，若卡片有被搜索的关键词，搜索结果高亮对应模块。


### 11.4 动效

使用 GSAP，但动效要克制。

允许：

- 页面入场
- 数字滚动
- 搜索高亮
- 弹窗打开和关闭
- KPI 小卡片翻转 / 放大为二级数据卡片

不允许：

- 大量无意义装饰动效
- 影响数字阅读的动效
- 让页面显得更乱的动效

## 12. UI 视觉要求

继续使用 `frontend-design` 的前卫设计方向，但整体需要更简洁。

整体配色（2026-06-24 确认）：**黑白极简**主题，普通文字和普通控件使用黑白；状态提示只使用**荧光绿 `rgb(223, 255, 0)` / 荧光粉**，不使用其它彩色渐变或其它绿色、黄色。

视觉关键词：

- 未来经营指挥舱
- 清晰分区
- 少卡片
- 少明细
- 大数字
- 强经营判断

设计目标：

- CEO 30 秒内能看懂。
- 一屏只讲月度经营。
- 信息密度可控。
- 避免复杂、拥挤、细碎。

## 13. 验收标准

Demo 需要满足：

- 页面能直接在浏览器打开。
- 首屏不出现明显重叠、裁切、拥挤。
- 关闭二级数据卡片后，能回到首屏当前月经营总览。
- ECharts 图表正常渲染。
- GSAP 动效正常但不干扰阅读。
- 搜索能高亮对应模块。
- 版本价格、渠道、财务指标展示清楚。
