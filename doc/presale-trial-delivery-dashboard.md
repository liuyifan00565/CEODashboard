# 售前试用交付看板数据口径

更新时间: 2026-07-14 15:11:44 CST
更新内容: 将交付看板整理为核心指标、六项月度对比带、渠道阶段概览和同屏双栏执行明细四层结构，数据口径保持不变。

## 页面定位与数据边界

“交付看板”保留现有侧边栏入口、菜单 key、权限与路由，页面内容重构为“售前试用转化与配置负载”。页面按以下顺序自然向下滚动：

1. 整体试用情况；
2. 六项本月与上月指标对比带；
3. 当前试用客户渠道占比与试用阶段风险双栏概览；
4. “交付执行明细”单一玻璃工作台，桌面端按 58% / 42% 同屏展示渠道 / 团队转化与配置人员负载。

当前数据库快照没有完整的售前试用生命周期字段，因此 `cockpit/src/data/presaleTrialDelivery.js` 继续集中维护按月演示快照，但这些快照仅供测试或显式注入加载器使用。`DeliveryPanel` 运行态默认使用稳定空加载器，真实售前字段未接入时展示空态；接入数据后，所有 KPI、环图、阶段、月度对比带和执行明细必须来自同一个 `monthKey` 快照，业务数值不得散落在组件内。

现有 `/api/dashboard-data` 契约保持不变，服务端仍可返回由 `fact_delivery_order` 与 `biz_delivery_target_monthly` 聚合的旧 `deliveryRows`，以兼容旧消费者和 getter。新售前试用页不读取、合并或回退到 `deliveryRows`，因为“已交付订单”不能替代“当前试用客户”；接口有旧数据、演示快照缺失时也必须展示空态或错误态，不能混出口径不一致的结果。

## 转化率 cohort

页面同时展示两个用途不同的 cohort，禁止互换分母：

- 核心 KPI 的试用转化率衡量“同一批进入试用的客户”：2026-07 为 `12 / 26 = 46.2%`，2026-06 为 `9 / 23 = 39.1%`。分子为该批已完成成交客户，分母为该批进入试用并已到统计观察点的客户。
- 渠道 / 团队表使用“上期启动且已完成观察窗的成熟队列”：2026-07 的渠道行分母分别为 12、10、8、6，成交分别为 7、3、4、1，合计 `15 / 36 = 41.7%`；2026-06 为 `12 / 32 = 37.5%`。
- 表内“当前试用”用于表达实时在试客户规模，不是转化率分母。环图的 60 个当前试用也不得参与成熟队列转化率计算。
- 团队视图只改变归属维度，不改变成熟队列定义。2026-07 四个团队的上期启动 / 已成交分别为 `10/6`、`9/3`、`11/5`、`6/1`，合计仍为 `15 / 36`。

## 风险分层

风险展示分为三层，避免把不同含义的数量直接相加：

- 阶段层：“已超期”是当前试用生命周期阶段之一。2026-07 共 5 个，和其它阶段互斥并共同组成 60 个当前试用客户。
- 核心介入层：“风险及超期 5 个”表示需要重点介入的 3 个临近到期客户与 2 个重点超期客户。2 个重点超期包含在阶段层的 5 个已超期客户中，不代表阶段超期只有 2 个。
- 数据治理层：未配置负责人、预计成交金额未填写等提醒表示资料完整性问题，可能与临近或超期客户重叠，不计入核心卡的简单加总。2026-07 提醒为 3 个将在 3 天内超过试用周期、2 个未配置负责人、3 个未填写预计成交金额。

2026-06 的“风险及超期 8 个”和提醒计数同样来自当月快照；提醒为 4 个临近、2 个未配置负责人、4 个预计金额缺失。提醒之间允许客户重叠，因此不要求提醒数量之和等于核心风险数。风险信息当前没有真实详情页，页面只展示状态图标、文字与层级箭头，不绑定无效跳转。

## 月度比较口径

月份选择器切换后，核心指标、月度对比带、环图、阶段、风险、转化明细和人员负载必须同时切换到相同月份。月度对比带始终使用所选月份与其自然上月：

- 普通增长率：`(本月 - 上月) / 上月 × 100%`，展示一位小数；
- 转化率：直接计算百分点差，单位为 `pp`，不能再除以上月转化率；
- 平均试用周期：直接计算天数差，单位为“天”；
- 新增、成交、转化率与金额上升为正向；风险数和平均试用周期下降为正向，均显示绿色“改善”。

2026-07 对 2026-06 的六项结果固定为 `+20.0%`、`+33.3%`、`+7.1pp`、`+20.1%`、`-37.5%`、`-1.6 天`。2026-06 对 2026-05 分别为 `+15.4%`、`+28.6%`、`+4.1pp`、`+12.1%`、`-20.0%`、`-0.9 天`。

## 配置人员负载

建议容量上限统一为 14 单，负载比例为 `当前负责客户数 / 14`。状态只由该比例派生：

- 小于 80%：正常；
- 大于等于 80% 且小于等于 100%：偏高；
- 大于 100%：超负荷。

按 14 单上限折算，当前负责 11 单及以下为正常，12 至 14 单为偏高，15 单及以上为超负荷。进度条表达实际比例，超出 100% 时状态仍为“超负荷”，视觉轨道可封顶显示但数值不能被改写。人员负载区不再展示客户均价、目标完成率或“目标未配置”。

## 加载、空数据与交互

- 接入或显式注入异步加载器后，初次进入和月份切换使用同一个加载器并展示骨架状态；旧月份响应不能覆盖较新的月份选择结果。
- 找不到完整月份快照时展示空数据状态；加载器抛错时展示接口异常状态和重试入口。
- 当前客户总数为 0、分布数组为空，或所选“客户数量 / 预计成交金额”维度合计为 0 时不初始化环图，避免出现无业务含义的 0% 饼图。
- 环图扇区或可聚焦的渠道图例点击后筛选“交付执行明细”左侧的渠道 / 团队转化区；再次点击同一渠道或关闭筛选标签即清除。切换团队视图时按预设渠道与团队映射保留筛选，切换月份时清除旧筛选。
- 图表 tooltip 同时显示渠道、客户数量、数量占比、预计成交金额和金额占比；数量 / 金额切换只改变扇区计算维度，不改变 tooltip 的完整字段。

## 后端待补字段

目前没有新增服务端查询或数据库迁移。若后续将演示数据替换为真实接口，至少需要以下稳定字段，并需保留同批 cohort 与成熟 cohort 的可追溯来源：

| 业务对象 | 必需字段 |
| --- | --- |
| 月份快照 | 业务月份、数据截至时间、快照生成时间、权限 / 组织范围 |
| 试用客户 | 唯一试用 ID、客户 ID、试用开始时间、计划结束时间、实际结束时间、当前阶段、是否超期、渠道 ID、团队 ID、配置负责人 ID |
| 转化 cohort | cohort ID 或进入试用月份、观察窗截止时间、是否进入成熟队列、成交状态、成交时间、关联订单 ID；核心 KPI 与渠道表必须能分别定位各自分母 |
| 金额 | 预计成交金额（建议以元存储）、已成交金额（元）、币种、预计金额是否已填写 |
| 渠道 / 团队 | 稳定 ID、显示名称、归属映射、当前试用数、上期启动数、成熟队列成交数、状态 |
| 风险 | 风险类型、严重级别、触发时间、临近到期天数、是否重点介入、是否缺负责人、是否缺预计金额；同一试用的多风险需保留关联 ID 以去重 |
| 配置人员 | 人员 ID、姓名、当前负责数、本月累计数、已转化数、超期数、名下预计金额、个人建议容量上限；没有个人值时才使用页面默认 14 单 |
| 月度比较 | 所选月及自然上月的新增试用数、同批成交数与分母、预计成交金额、风险及超期数、平均试用周期；原始值由前端统一计算变化值 |

真实数据接入时还需要后端明确阶段枚举、超期判定时区、成交去重规则、客户跨渠道归属规则、金额含税 / 币种口径，以及权限过滤后各模块合计是否仍可闭合。在这些字段和规则齐备前，不得用旧 `deliveryRows` 推算售前试用指标。

## 一致性验收

- 所选月份的当前试用总数、渠道数量合计与阶段合计必须一致；2026-07 均为 60，2026-06 均为 52。
- 渠道预计金额合计必须等于核心预计成交金额；2026-07 为 82.4 万，2026-06 为 68.6 万。
- 2026-07 人员当前负责 58 加未配置负责人 2 等于当前试用 60。
- 月度比较必须由原始值计算，转化率使用 `pp`，风险数和周期下降显示为改善。
- 1440px 与 1920px 桌面宽度下，月度对比带保持六列，交付执行明细保持 58% / 42% 双栏；不得出现页面级横向滚动或模块重叠。
- 宽度不超过 1200px 时，月度对比带改为三列两行、执行明细上下堆叠；宽度不超过 760px 时，对比带改为两列三行，明细指标自动换行且业务数值不可被截断。
