## 模块位置

经营总览顶部区域由「月度回款主卡」和「年度回款总览卡」组成，独立的「渠道目标完成结构」模块不再单独占位。

月度主卡内部由三部分组成：

- 左侧：本月回款、月度目标、月目标完成率、超额完成/目标缺口。
- 中间：本月回款结构半环图。
- 右侧：各渠道经营情况，展示实际回款、目标回款、完成率、缺口与风险标签。

年度回款总览卡借用月度主卡的三栏骨架，但视觉权重略低于本月回款：

- 左侧：年度累计回款大数字、年度目标、年目标完成率、剩余目标。
- 中间：年度回款结构半环图，按年度实际回款拆分渠道占比。
- 右侧：年度经营情况，展示各渠道年度回款、年度目标、完成率、缺口与风险标签。
- 年度目标进度条位于年度左侧事实区两个胶囊下方，桌面端向右延展到接近右侧经营情况列左侧留白，显示 `年度累计回款 / 年度目标`、进度条和完成率。

年度右侧经营情况列独占第三列，年度进度 footer 不进入经营情况列，避免压缩或覆盖经营列表。

年度进度条在桌面端从左侧事实区向右延展，靠近两个目标状态胶囊；中等和窄屏布局下恢复为左侧事实区整行宽度，避免横向溢出。

月度与年度右上角下钻入口统一改为文字箭头形式：`点击查看近期明细 ›`、`点击查看年度拆解 ›`。入口本身保持较大的可点击热区，但视觉上不再呈现为胶囊按钮。

## 数据来源

- 月度主卡继续读取 `KPI.monthRecovered`、`KPI.monthTarget` 和 `KPI_DERIVED.monthCompletion`。
- 渠道行数据继续使用前端统一方法 `getChannelCompletionRows('month')`。
- 年度总览读取 `KPI.yearRecovered`、`KPI.yearTarget`、`KPI_DERIVED.yearCompletion` 和 `getChannelCompletionRows('year')`。
- 渠道实际回款读取行内 `recovered`，目标回款读取行内 `target`，完成率读取行内 `completion`。
- 渠道缺口 = `max(0, target - recovered)`。
- 风险标签优先沿用渠道行 `warn` / `status` 结果；若渠道完成率低于当前整体完成基准，也标记为风险渠道。
- 当前整体完成基准 = `min(100%, 本月整体完成率)`。例如本月整体完成率 106.0% 时，完成率低于 100% 的渠道标记风险。

## 图表口径

半环饼图表达的是「当前周期目标口径下的回款构成」，由各渠道实际回款和未完成目标缺口共同组成。

- 月度卡各渠道扇区 = 当前渠道本月实际回款 / `max(月度目标, 本月实际回款)`。
- 年度卡各渠道扇区 = 当前渠道年度实际回款 / `max(年度目标, 年度实际回款)`。
- 未完成扇区 = `max(0, 当前周期目标 - 当前周期实际回款)`，以低透明灰蓝扇区退到背景里，只作为目标缺口占位。
- 半环中心不显示数字，避免与左侧 `月目标完成率` 重复。
- 月度目标是否达成由左侧主数字事实胶囊表达，年度目标节奏由左侧事实胶囊和底部进度 footer 表达。
- 当当前周期实际回款为 0 时，图表使用低透明占位扇区，避免空图造成布局跳变。

## 右侧经营情况

右侧每个渠道展示：

- 渠道名称
- `实际 x万 / 目标 y万`
- 完成率
- 缺口金额，仅在未达目标时显示
- 风险标签，仅在渠道低于风险基准时显示

贡献占比只通过半环图和 tooltip 表达，右侧列表不再重复展示「贡献 x%」，避免经营列表从行动信息退化成图例。

## 目标状态与风险

月度目标状态跟随本月回款主数字展示：

- 当本月回款低于月度目标时显示「目标缺口 x万」。
- 当本月回款达到或超过月度目标时显示「超额完成 x万」。

不再单独显示「风险渠道」提醒卡。渠道风险只在右侧经营情况行内通过淡红背景、红色弱边框和小标签表达，具体缺口也在对应渠道行内查看，避免风险提示覆盖「本月回款」主结论。

## 年度目标进度

年度左侧事实区进度条表达全年节奏：

- 主进度 = `KPI.yearRecovered / KPI.yearTarget`，宽度上限为 100%。
- 左侧文字只展示 `年度累计回款 / 年度目标`，不再显示可见的 `年度目标进度` 标题。
- 右侧展示年目标完成率。
- 进度条下方不再展示时间进度、线性进度差或后续月均需完成金额，避免年度卡底部信息过密。

该进度条从年度左侧事实区内联渲染，桌面端只向前两列方向延展，不改变年度经营情况列的可用宽度。
