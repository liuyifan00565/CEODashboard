## 模块位置

经营总览顶部区域依次由「年度回款总览折叠条」和「月度回款主卡」组成，独立的「渠道目标完成结构」模块不再单独占位。年度折叠条与月度主卡保持相同宽度，默认仅展示标题、年度目标进度和完成率，最右侧向下箭头用于展开年度详情。

年度详情收起时优先为下方经营信息释放首屏高度；展开后，月度和年度回款结构饼图共用相同尺寸，桌面端图区高度为 200px、图表宽度为 380px 到 480px；低有效高度桌面档统一为 190px 和 360px 到 440px。回款结构区相对三栏分界线轻微左移，以减少中间结构和右侧经营情况之间的挤压感。

总览主卡下方先展示两张同级图表：左侧为月度经营趋势，右侧为成交来源环形图。来源环图只读取数据库最新业务月，与首页月份保持一致；环图在卡片内居中，中心显示该月净回款总额，每个扇区通过外部名称、占比和引导线直接标注，不再使用右侧图例。来源超过 7 个时其余来源合并为「其他来源」扇区。点击环形图或卡片下方明细入口后，弹出同一业务月的完整来源排行，逐项展示净回款、成交笔数、客户数和贡献占比。

两张图表下方展示四张等宽经营指标卡：本月开户数、今日开户数、总投入费比和广告 ROI。窄屏时四张指标卡改为两列或单列，趋势与成交来源图改为上下排列。

月度主卡内部由三部分组成：

- 左侧：本月回款大数字、月度目标、月目标完成率、超额完成/目标缺口；大数字上方不再重复显示“本月回款”小标签。
- 中间：本月回款结构半环图。
- 右侧：各渠道经营情况，展示实际回款、目标回款、完成率、缺口与风险标签。

年度回款总览默认收起为横向概览条：左侧为「年度回款总览」标题，中间为 14px 年度目标进度条和右侧完成率，最右为展开箭头。点击箭头后显示借用月度主卡骨架的三栏详情：

- 左侧：年度累计回款大数字、年度目标、年目标完成率、剩余目标。
- 中间：年度回款结构半环图，按年度实际回款拆分渠道占比。
- 右侧：年度经营情况，展示各渠道年度回款、年度目标、完成率、缺口与风险标签。
- 年度目标进度条始终位于折叠条内，外围不使用独立胶囊底板，也不重复显示累计/目标数值；展开详情后不再显示第二条重复进度。

年度详情中的右侧经营情况列独占第三列，折叠条进度位于详情外部，不压缩或覆盖经营列表。窄屏下标题和箭头保持同一行，进度条与完成率换到下一行，避免横向溢出。

全局搜索从主内容右上角移到左侧导航下方，以 44px 紧凑玻璃行呈现；主内容取消独占搜索顶栏并与侧栏顶部对齐，释放约 50px 首屏高度。

月度下钻入口 `点击查看近期明细 ›` 位于本月回款结构半环图右下方，桌面三栏布局下首字“点”对应饼图右半区起点标线、底边对齐月度经营情况的线下华东行底边；年度入口 `点击查看年度拆解 ›` 在展开后位于年度回款结构半环图右下区域。两个入口都保持文字箭头形式，并在文字周边保留透明点击热区；用户没有精准点中文字时，点到入口附近也会进入对应 KPI 二级页面，不呈现为胶囊按钮。

## 数据来源

- Excel 自营收入文件只写入实际收入、退款、人员、客户、订单和来源等事实字段，不生成或猜测目标。
- 当前回款 KPI、月度趋势和渠道结构以「福客2026年4-6月业绩」的月度大数为准，不再使用 1-4 月订单明细参与聚合。
- 南棠渠道金额并入「代理」渠道实际回款；退款仍使用代理退款列。
- 特殊渠道已包含在月度总额中，另以 `record_level = structure` 供半环拆分；它不关联 `dim_channel`，不出现在右侧经营渠道、ROI、维护页或人员下钻中。
- 成交来源读取 `fact_revenue_order.channel_source_id` 关联 `dim_channel_source`；维表名称缺失时回退到订单原始来源字段 `channel_name_raw`，仍缺失时显示「未标注」。
- 成交来源排行只统计数据库最新业务月的净回款；净回款为销售业绩减退款。每个来源同时返回成交笔数、去重客户数、首末成交日期，并可跟随侧栏经营渠道筛选。
- 来源贡献占比 = 当前来源净回款 / 当前筛选范围内全部来源净回款。列表按净回款降序，首页最多展示前 8 个来源，完整来源数量显示在面板右上角。
- 月度、年度及渠道目标独立读取 `biz_target_monthly` 的部门级配置；人员目标只有数据库存在人员分配时才可展示，不能由实际收入反推。
- 目标维护保存或导入成功后，`MaintenancePage` 通知 `App` 重新请求 `/api/dashboard-data`，回到经营总览时直接显示最新目标。
- 月度主卡继续读取 `KPI.monthRecovered`、`KPI.monthTarget` 和 `KPI_DERIVED.monthCompletion`。
- 渠道行数据继续使用前端统一方法 `getChannelCompletionRows('month')`。
- 年度总览读取 `KPI.yearRecovered`、`KPI.yearTarget`、`KPI_DERIVED.yearCompletion` 和 `getChannelCompletionRows('year')`。
- 渠道实际回款读取行内 `recovered`，目标回款读取行内 `target`，完成率读取行内 `completion`。
- 渠道缺口 = `max(0, target - recovered)`。
- 风险标签优先沿用渠道行 `warn` / `status` 结果；若渠道完成率低于当前整体完成基准，也标记为风险渠道。
- 当前整体完成基准 = `min(100%, 本月整体完成率)`。例如本月整体完成率 106.0% 时，完成率低于 100% 的渠道标记风险。

## 图表口径

月度经营趋势中的完成率是核心指标，折线使用偏亮银紫色 `#B9B6E8`、2.8px 线宽、90% 透明度、0.25 平滑度和 6px 轻外发光；常规数据点为 7px，低于 80% 的风险月份放大为 9px。百分比文字始终保留，仅低于 80% 时使用风险红，94.5% 等正常完成率继续使用银紫色。

半环饼图表达的是「当前周期目标口径下的回款构成」，由各渠道实际回款、特殊渠道展示金额和未完成目标缺口共同组成。

- 月度卡各渠道扇区 = 当前渠道本月实际回款 / `max(月度目标, 本月实际回款)`。
- 年度卡各渠道扇区 = 当前渠道年度实际回款 / `max(年度目标, 年度实际回款)`。
- 未完成扇区 = `max(0, 当前周期目标 - 当前周期实际回款)`，以低透明灰蓝扇区退到背景里，只作为目标缺口占位。
- 半环中心不显示数字，避免与左侧 `月目标完成率` 重复。
- 月度目标是否达成由左侧主数字事实胶囊表达，年度目标节奏由左侧事实胶囊和底部进度 footer 表达。
- 当当前周期实际回款为 0 时，图表使用低透明占位扇区，避免空图造成布局跳变。

## 右侧经营情况

右侧每个渠道展示：

- 渠道名称
- `实际 x万 / 目标 y万`
- 完成率
- 缺口金额，仅在未达目标时显示
- 风险标签，仅在渠道低于风险基准时显示

渠道行之间保留清晰的垂直间隔：桌面默认 16px，低有效高度桌面档 13px，避免四个经营情况条目在视觉上贴成一组。

贡献占比只通过半环图和 tooltip 表达，右侧列表不再重复展示「贡献 x%」，避免经营列表从行动信息退化成图例。

## 目标状态与风险

月度目标状态跟随本月回款主数字展示：

- 当本月回款低于月度目标时显示「目标缺口 x万」。
- 当本月回款达到或超过月度目标时显示「超额完成 x万」。

不再单独显示「风险渠道」提醒卡。渠道风险只在右侧经营情况行内通过淡红背景、红色弱边框和小标签表达，具体缺口也在对应渠道行内查看，避免风险提示覆盖「本月回款」主结论。

## 年度目标进度

年度左侧事实区进度条表达全年节奏：

- 主进度 = `KPI.yearRecovered / KPI.yearTarget`，宽度上限为 100%。
- 左侧删除大数字上方重复的 `年度累计回款` 小标签，仅保留回款大数字、`年度目标` 和目标状态信息；不显示可见的 `年度目标进度` 标题。
- `点击查看年度拆解` 入口放在 `年度回款结构` 半环图右下区域、年度进度条上方；定位保持贴近半环右下，不落到进度条下边。
- 右侧展示年目标完成率。
- 进度条下方不再展示时间进度、线性进度差或后续月均需完成金额，避免年度卡底部信息过密。

该进度条从年度左侧事实区内联渲染，桌面端只向前两列方向延展，不改变年度经营情况列的可用宽度。
