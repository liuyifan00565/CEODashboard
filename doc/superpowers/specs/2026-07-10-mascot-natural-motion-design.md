# 福小客自然丝滑动作优化设计

## 目标

在不更换福小客形象、不改变侧栏布局和点击交互的前提下，让左下角小人的待机、挥手、指引、说话、思考、提醒、庆祝和点击动作更自然、更有生命感。

本次优化解决以下问题：

- 逐帧播放接近匀速，动作缺少起势、主动作、停顿和回落的层次。
- 待机循环较短，呼吸和眨眼容易显得机械重复。
- 循环动作在端点立即反向，观感生硬。
- Sprite fallback 与本地 rig 分别计算播放进度，存在观感不一致的风险。
- 动作切换虽然避免了双影，但当前离散帧 bridge 仍可能出现突然换姿势。

## 设计原则

1. 保持人物尺寸、脚底位置和侧栏入口边界稳定。
2. 不使用人物缩放或大幅旋转掩盖素材问题。
3. 外层连续动效只允许约 1–2px 的轻微位移。
4. 不恢复旧帧叠加或透明交叉淡化，避免出现双人重影。
5. 所有动作使用同一套时间线计算规则，Sprite fallback 与本地 rig 共享结果。
6. 尊重 `prefers-reduced-motion`，减少动态时停留在清晰稳定的代表帧。

## 动作时间线

### 待机

- 一个完整循环控制在约 5–6 秒。
- 大部分时间保持自然站立，只加入缓慢、不对称的呼吸起伏。
- 每个循环只出现一次短眨眼；闭眼帧短暂停留，避免长时间闭眼。
- 循环首尾使用相同或视觉接近的开放眼姿态，避免回到首帧时跳动。

### 一次性动作

挥手、指引、庆祝和点击统一分成四个阶段：

1. 起势：短促但不突然，快速进入可识别姿势。
2. 主动作：动作幅度最明确。
3. 短停：在语义最清楚的帧停留片刻。
4. 回落：沿原动作帧反向或使用收势帧回到稳定状态。

帧间时长不再全部相同；起势稍快，主动作自然，短停最慢，回落柔和。

### 循环动作

说话、思考、提醒和维护复核使用柔和往返时间线：

- 正向播放至动作端点。
- 端点短暂停顿后再反向。
- 反向结束时再保留一个短呼吸停顿，避免连续机械摆动。
- 提醒动作可以比说话和思考稍快，但位移上限保持一致。

## 共享播放模型

在 `mascotAnimationManifest` 中为每个动作声明明确的帧时间线，而不是只声明统一 `fps`。时间线由帧编号和对应持续时间组成，例如：

```js
timeline: [
  { frame: 0, durationMs: 120 },
  { frame: 1, durationMs: 90 },
  { frame: 2, durationMs: 90 },
]
```

新增纯函数，根据已播放毫秒数解析当前帧、循环次数和是否结束。`MascotSpriteStage` 与 `FuxiaokeLocalRigStage` 都调用该函数，从而消除两套匀速播放实现。

时间线函数不依赖 React 或浏览器，便于对边界、循环和减少动态模式进行确定性测试。

## 动作衔接

动作切换继续使用单层画面，不创建 ghost、blend 或 crossfade 图层。

切换规则如下：

- 当前动作拥有收势阶段时，先播放剩余收势帧。
- 下一动作从起势阶段进入，不重复播放已经用于 bridge 的开头帧。
- 从待机进入动作时直接使用下一动作的短起势，不额外倒放待机帧。
- 新动作在 bridge 期间再次到来时，以最新动作作为目标，避免排队播放过时动作。
- `prefers-reduced-motion` 开启时跳过 bridge，直接显示目标动作的代表帧。

由于现有素材不是正式 Cubism 骨骼模型，本设计不承诺不同姿势之间产生真实形变插值；目标是在现有帧素材边界内消除匀速、顿挫、过度眨眼和重复反向等机械感。

## 外层连续动效

`MascotSpriteStage.css` 与 `Live2DMascotStage.css` 的生命感动效保持脚底锚定：

- 待机使用慢速、非对称的纵向位移，最大约 1.2px。
- 说话使用幅度更小、节奏稍活跃的位移。
- 思考使用更慢的偏移和较长停顿。
- 挥手、指引和庆祝只在一次性动作期间出现短位移，结束时回到 `translate3d(0, 0, 0)`。
- 不使用 `scale`，不增加大幅 `rotate`，避免人物忽大忽小或脚底漂移。

## 兼容与降级

- 本地 rig 可用时优先显示本地 rig。
- rig 不可用时 Sprite fallback 使用完全相同的动作时间线。
- 动作表、模型入口或 motion 文件不可用时继续沿用现有 fallback 规则，不隐藏小人入口。
- 减少动态模式下不启动持续动画，并显示动作语义清晰的静态帧。

## 测试与验收

### 自动测试

- 验证所有运行时动作都声明合法时间线，帧编号不越界，持续时间为正数。
- 验证待机总时长处于 5–6 秒范围，闭眼帧只占短暂比例。
- 验证一次性动作包含起势、主动作、停顿和回落，最终稳定结束。
- 验证循环动作端点有停顿，且往返序列不会重复造成抽动。
- 验证 Sprite fallback 与本地 rig 使用同一个时间线解析函数。
- 验证减少动态模式不启动逐帧循环或 bridge。
- 保留现有素材尺寸、安全边距、脚底抖动和动作差异审计。

### 运行验收

- 左下角人物尺寸与当前一致，脚底位置稳定。
- 待机观察至少两个循环，无频繁眨眼、突然回首帧或忽大忽小。
- 依次触发挥手、点击、说话、思考、提醒和庆祝，动作语义清楚。
- 连续快速切换动作时不出现双影、空白帧或过时动作排队。
- 切换页面和打开 AI 对话框的原有行为保持不变。
