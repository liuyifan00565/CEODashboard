🔴 被拦路径（skipped 拒绝）

四种拒绝场景，written=0，给出明确原因
1
非销售：赵晴 is_sales=0 → 拒

2
无部门：人员 department_id=NULL → 拒

3
停用：人员 is_enabled=0 → 拒
XX 已停用，无法导入目标。请先在组织维护页启用该人员
4
部门不符：王丽英写华南战区(实为线上部) → 拒
王丽英 不属于「华南战区」，实际属于「线上销售部」，跳过
5
旧模板：无「所属组织」列 → 整文件拒
必填列【所属组织】未在 Excel 表头中找到 → accepted=0, written=0
口径已统一：导入、页内保存、列表显示和首页目标分母都认「is_sales=1、is_enabled=1 且有部门」。无部门/非销售/停用人员既看不到也导不进，旧目标也不会继续进入看板分母。

经营总览 本月回款、月目标完成率、目标缺口、风险渠道、年度节奏。
渠道完成情况 本月/年度切换，按渠道展示完成、目标、完成率和缺口。
月度趋势 按月展示目标 vs 回款，渠道筛选时按渠道比例/渠道数据切换。

开户 / 版本 / 续费 开户实际、版本销量、版本续费到期与已续。
交付 / 算力 交付订单、工程师目标；算力消耗、余额、版本消耗和资源健康。
1. 前端加载 loadDashboardData() 请求 GET /api/dashboard-data，禁止缓存，拿到真实 MySQL 快照。
2. 后端聚合 buildDashboardSnapshot() 同时查询事实表、维护表和维表，再组装统一 snapshot。
3. 覆盖运行时数据 applyDashboardDataSnapshot() 把 KPI、渠道、趋势、版本、算力等数组替换为数据库结果。
4. 主界面渲染 OperatingOverview、MonthlyTrend、KpiCard、VersionFinancePanel 等组件直接消费运行时数据。
重点：维护页保存后会重拉维护页自身数据；主界面看到新口径，需要回到主界面后重新加载 /api/dashboard-data。当前代码没有单独的“只刷新某张卡片”接口。

⑥ 主界面模块 ↔ 数据库表 / 维护页

主界面区域主要读哪些表关键计算口径受哪个维护页影响备注
经营总览：本月/年度回款、完成率、缺口 fact_revenue_dailybiz_target_monthlydim_staff 实际回款按日期聚合；目标只统计 启用销售+有部门 人员的 target_amount_yuan。 目标维护组织维护 组织页停用、转非销售、移除部门后，旧目标不再进入首页目标分母。
渠道完成情况 / 风险渠道 dim_channelfact_revenue_dailybiz_target_monthlydim_staff.channel_key 渠道实际来自回款事实表；渠道目标来自销售人员目标按 channel_key 汇总。 目标维护组织维护渠道维护 只展示启用渠道。渠道大类/名称维护会影响展示维度；销售人员所属渠道影响目标归因。
月度趋势 biz_target_monthlyfact_revenue_daily 每月目标与回款按月份对齐；当前月目标优先使用维护表汇总值。 目标维护组织维护 目标改动后会直接改变柱状图目标线/目标柱。
总投入 · 费比 / 渠道 ROI biz_channel_cost_monthlybiz_labor_cost_monthlyfact_revenue_dailydim_channel 总投入 = 渠道广告成本 + 人力成本；费比 = 总投入 / 本月回款；渠道 ROI = 渠道回款 / 渠道成本。 成本维护渠道维护 成本页新年份默认给销售/市场人力成本行，避免首页费比缺少人力项。
开户卡片 fact_opening_account_daily 本月开户、今日开户和环比/较昨日来自实际开户事实表。 暂不由四个维护页直接改 目标维护导入会写开户/订单目标，但当前首页开户卡主要展示实际值和变化率。
版本销售 / 续费面板 fact_version_sales_dailyfact_renewal_dailydim_product_versiondim_channel 版本销量、回款、环比、当月应续/已续按版本和渠道聚合。 渠道维护 渠道维表影响渠道名称/筛选；版本本身暂不在四个维护页维护。
交付看板 fact_delivery_orderbiz_delivery_target_monthlydim_staff 交付数按订单计数；交付目标按工程师和月份读取。 暂不由四个维护页直接改组织维护影响姓名维表 如果后续要让交付目标可维护，应单独加交付目标维护入口或导入配置。
算力用量分析 fact_compute_usage_dailyfact_compute_customer_dailyfact_compute_version_consumption_dailyfact_compute_resource_health_dailydim_compute_* 算力消耗、充值、容量、客户余额、版本消耗分布和资源健康来自算力事实表。 不受四个维护页影响 这块由同步/采集链路维护，不走目标、成本、组织、渠道四页。
目标维护会改什么 月/年目标、完成率、缺口、月度趋势目标、渠道目标、风险渠道排序。
成本维护会改什么 总投入、费比、广告/人力拆分、渠道 ROI 和成本占比。
组织维护会改什么 哪些人员进入目标分母；人员姓名、销售身份、启停和部门归属。
渠道维护会改什么 首页渠道列表、渠道名称/层级、后续线索来源归因和启用状态。
