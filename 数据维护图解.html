② 四条数据怎么流

🔵 加载（读）GET /api/maintenance/data

页面→后端 reader→MySQL 查
原始行→mapper 组装→渲染表格
触发：切页 / 改年份 / 导入或保存后重拉。带 year 和 pageKey 两个参数。

🟢 页内编辑保存（写）POST /api/maintenance/save

表格草稿→collect()→部分列 upsert→MySQL 写
写完→重拉回显
只写本页可编辑的列，不覆盖未编辑列。金额 万→元(×10000)。

🟡 Excel 导入（写）POST /api/maintenance/import

选 xlsx→解析校验→再校验+upsert→MySQL 写
written>0→重拉
前端解析一遍，后端用原始行再解析一遍。FK 对不上就跳过。

🔎 下载模板纯前端

配置→生成 xlsx→浏览器下载
按 maintenanceImportConfig.js 生成表头+示例+说明 sheet。

④ 现在的防护：非销售/无部门 → 导入直接被拒

🟢 正常路径（写入成功）

Excel: 王丽英 / 所属组织=线上销售部 / 2026-08 / 回款100 / 开户5 / 订单3
1
按姓名 王丽英 查 dim_staff → 找到 staff_id=2001
SELECT s.*, d.department_name FROM dim_staff s LEFT JOIN dim_department d ... WHERE staff_name='王丽英' → is_sales=1, dept=线上销售部 ✓
2
校验：是销售 ✓ / 有部门 ✓ / Excel部门=实际部门 ✓
3
upsert 写入 biz_target_monthly → written=1 ✅
模板必须有「所属组织」列，且与人员实际部门一致。

🔴 被拦路径（skipped 拒绝）

三种拒绝场景，written=0，给出明确原因
1
非销售：赵晴 is_sales=0 → 拒
赵晴 不是销售，无法导入目标。请先在组织维护页将其设为销售
2
无部门：人员 department_id=NULL → 拒
XX 无所属组织，无法导入目标。请先在组织维护页分配部门
3
部门不符：王丽英写华南战区(实为线上部) → 拒
王丽英 不属于「华南战区」，实际属于「线上销售部」，跳过
4
旧模板：无「所属组织」列 → 整文件拒
必填列【所属组织】未在 Excel 表头中找到 → accepted=0, written=0
口径已统一：导入、页内保存、列表显示三处都认「is_sales=1 且有部门」。无部门/非销售的人员既看不到也导不进。

⑤ 这套界面的口径坑（#1 已修复）

✓
目标页：导入不校验 is_sales，显示只看 is_sales=1 —— 已修复(2026-07-08)。导入现在校验「是销售+有部门+部门一致」，非销售/无部门直接拒；列表也加「有部门」过滤；页内保存同步加门槛；孤儿目标已清。
2
目标/成本页：导入不校验月份年份，显示按 year 过滤 —— 导 2025 数据、页面停 2026，重拉也查不到。
3
导入按姓名精确匹配 FK（无模糊） —— Excel 名字带空格/全半角差异就跳过，written 可能=0 但看着像"导完了"。
4
页内保存只写金额，不写开户/订单目标 —— saveTarget 只更新 target_amount_yuan，开户数/订单数只能靠导入。
5
目标页组织树人数只数 is_sales=1 且 is_enabled=1 —— 和组织页全量人员口径不同，对不上。
