## Goal

把当前 dashboard 从偏普通深色驾驶舱升级为克制、高级的 Apple / Vision Pro 风格经营驾驶舱。品牌主色统一为冷紫和蓝紫，背景使用 ReactBits Color Bends 作为可感知的亮紫带状环境光，而不是被遮罩吃掉的低存在感色块。

## Confirmed Direction

采用 B 方案：冷紫液态玻璃。

Color Bends 应占据全屏背景，并能明显看出 ReactBits 参考组件的亮紫带状流光。组件背景层应完整绘制，并允许穿透页面和半透明玻璃卡片；但亮度必须受控，不能出现白紫光束直接洗掉数据内容。形态必须是连续、弯曲、有方向的丝带状 bends，而不是普通紫色雾光或大块径向光斑。基底仍使用深海蓝黑 `#070A12`、`#0B1020`、`#121933`，流光只使用紫色系和浅紫高光。动效必须非常慢，像系统暗色背景后的液态环境光。

## Visual Rules

- 不使用彩虹色、多色混搭、点阵、网格、粒子、星空、科技线条。
- 紫色避免高饱和荧光紫、玫红紫、电竞紫和赛博霓虹。
- 背景之上叠加深色遮罩和暗角，优先保证卡片和数据可读。
- 前景卡片复用现有玻璃体系：透明背景、`var(--line-2)`、`var(--glass-blur)`、`var(--glass-shadow)`。
- hover 只做轻微抬起、边框变亮和很淡的蓝紫环境光。
- 进度条使用 `#7C6CFF -> #A79CFF -> #C9C2FF`，尾部可少量 `#9EDCFF`。
- 预警项使用柔和粉紫 `#F08AC3 -> #F5A4CF`，不使用高饱和警示光。
- 半环图围绕品牌紫系展开，未完成段使用低透明灰蓝紫。

## Implementation Scope

本次只改视觉层，不改数据口径、接口、同步逻辑、表结构或业务计算方式。

主要涉及：

- 新增 ReactBits Color Bends 背景组件，替换当前 Silk 背景入口。
- 调整全局主题变量、玻璃卡片、搜索高亮、进度条、KPI 半环和版本半环配色。
- 更新现有静态测试，确保关键颜色、背景组件和反霓虹约束可回归。
