# 成本维护隐藏赢单金额展示设计

## 背景

用户原始需求：

> 成本维护那个板块我想要把赢单全部删除了，不想要他在我的界面显示了

已确认的需求边界：

- 只删除成本维护表格单元格中可见的“赢单 + 金额”展示行。
- 保留成本、成交、退款、ROI 展示。
- 不删除后端赢单数据字段，不修改接口返回结构。
- 不修改 ROI 计算。ROI 仍然使用原有赢单金额和成本数据计算，只是不在该单元格里直接展示赢单金额。

## 推荐方案

采用最小前端展示调整：

1. 在 `cockpit/src/components/MaintenancePage.jsx` 的成本维护周期单元格中，移除 `赢单 {formatWan(period.actual)}` 这一行 JSX。
2. 保留同一单元格内的成本编辑、成交单数、退款编辑、ROI 展示。
3. 在文件顶部新增本次更新时间和更新内容说明。
4. 同步补充测试，防止成本维护页面后续又显示“赢单”文本。
5. 同步更新维护页内容说明文档，记录成本维护界面展示口径已调整。

## 不采用的方案

- 不用 CSS 隐藏。“赢单”仍会存在于 DOM 中，后续测试和辅助读取仍可能看到，不符合“不要在界面显示”的目标。
- 不删后端 `actual` 字段。该字段仍被 ROI 和其他统计口径使用，删除会扩大影响范围。
- 不改成本维护表格结构和样式。当前需求只是删除一个展示项，不需要调整玻璃样式、列宽、固定列或滚动逻辑。

## 影响范围

计划修改：

- `cockpit/src/components/MaintenancePage.jsx`
  - 删除成本维护单元格内“赢单”展示行。
  - 添加本次更新注释。
- `cockpit/src/App.layout.test.js`
  - 增加或调整回归测试，断言成本维护页面不再渲染“赢单”文本，同时保留 ROI 相关展示。
- `docs/superpowers/specs/2026-07-02-maintenance-pages-content-design.md` 或对应维护页说明文档
  - 记录成本维护显示项从“成本、赢单、成交、退款、ROI”调整为“成本、成交、退款、ROI”。

不修改：

- `cockpit/server/maintenanceMappers.js`
- `cockpit/server/maintenanceData.js`
- 成本维护保存接口
- 数据库表结构
- ROI 计算公式

## 验证方式

实现后执行以下验证：

1. 运行前端相关测试，确认成本维护页面不再显示“赢单”且 ROI 仍存在。
2. 搜索成本维护渲染路径，确认没有新的可见“赢单”展示行。
3. 若本机 Docker 可用，按项目规则重启对应服务并通过页面或接口验证；若 Docker 仍受主机虚拟化限制，则记录该环境阻塞，并用本地测试/构建作为代码级验证。

## 成功标准

- 用户进入成本维护页面后，看不到“赢单”这一行及其金额。
- 成本、成交、退款、ROI 仍按原布局显示。
- 数据计算口径不变，ROI 没有因为隐藏展示而变化。
- 本次提交只包含该展示调整、对应测试和文档，不混入已有目标维护未提交改动。
